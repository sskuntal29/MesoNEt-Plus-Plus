# MesoNet++: Robust Deepfake Detection via Shallow CNNs

> A robustness-focused extension of [MesoNet](https://arxiv.org/abs/1809.00888) with Squeeze-and-Excitation attention, JPEG augmentation, and label smoothing — achieving **93.6% accuracy** and a **70% reduction in JPEG compression degradation** over the Meso-4 baseline.

## Overview

Deepfake detectors that perform well in lab conditions often fail in the wild. Compressed social media images, brightness shifts, and unseen GAN architectures can cause accuracy to collapse. MesoNet++ addresses these failure modes through targeted, principled modifications to the lightweight MesoNet family — without sacrificing the compact parameter footprint that makes it deployable.

**Key results on the 140k Real and Fake Faces dataset (FFHQ vs. StyleGAN2):**

| Model | Accuracy | AUC | Params | JPEG-50 Drop |
|---|---|---|---|---|
| Meso-4 (baseline) | 94.6% | 0.987 | 27,977 | −4.7 pp |
| MesoInception-4 (baseline) | 92.1% | 0.975 | 28,615 | −3.8 pp |
| **MesoNet++ (ours)** | **93.6%** | **0.984** | **~32,000** | **−1.4 pp** |

## Architecture

MesoNet++ builds on the MesoNet encoder with four stages, each followed by a **Squeeze-and-Excitation (SE) block** for adaptive channel recalibration:

```
Stage 1: Inception(3 → 9 ch)  → SE(9)   [256×256 → 128×128]
Stage 2: Inception(9 → 8 ch)  → SE(8)   [128×128 →  64×64]
Stage 3: Conv5×5(8  → 16 ch)  → SE(16)  [ 64×64  →  32×32]
Stage 4: Conv5×5(16 → 32 ch)  → SE(32)  [ 32×32  →   8×8 ]

Classifier: Flatten(2048) → FC(64) → BN → LeakyReLU → Dropout(0.5)
                          → FC(32) → BN → LeakyReLU → Dropout(0.3)
                          → FC(1)
```

### Improvements over Meso-4

| Component | Addresses | Description |
|---|---|---|
| SE blocks after every encoder stage | JPEG robustness, attention | Channel-wise recalibration amplifies forgery-discriminative features |
| JPEG compression augmentation | JPEG robustness | Random JPEG re-encoding at quality ∈ [40, 95] applied with 50% probability during training |
| Label Smoothing BCE (ε = 0.1) | Overconfidence / calibration | Soft targets prevent logit saturation; ECE = 0.0089 |
| Widened Conv4 (16 → 32 channels) | Perturbation robustness | Richer feature representations for diverse corruption types |
| Deeper classifier head | Perturbation robustness, calibration | Three-layer FC head with progressive BatchNorm and Dropout |
| AdamW + OneCycleLR | Generalisation | 10% warmup + cosine annealing; gradient clipping at norm 1.0 |

## Dataset

Experiments use the [140k Real and Fake Faces](https://www.kaggle.com/datasets/xhlulu/140k-real-and-fake-faces) dataset:

- **Real:** ~70,000 images from [FFHQ](https://github.com/NVlabs/ffhq-dataset)
- **Fake:** ~70,000 images generated by [StyleGAN2](https://github.com/NVlabs/stylegan2)
- All images pre-cropped and resized to 256×256

| Split | Real | Fake | Total |
|---|---|---|---|
| Train | 50,000 | 50,000 | 100,000 |
| Validation | 10,000 | 10,000 | 20,000 |
| Test | 10,000 | 10,000 | 20,000 |

## Results

### JPEG Compression Robustness

MesoNet++ degrades only **1.4 percentage points** at JPEG quality 50, versus 4.7 pp for Meso-4 — a **70% improvement**.

### Perturbation Robustness

| Perturbation | Meso-4 | MesoInception-4 | MesoNet++ |
|---|---|---|---|
| None (baseline) | 94.6% | 92.1% | 93.6% |
| Brightness +0.3 | 91.2% | 89.4% | 92.1% |
| Brightness −0.3 | 90.8% | 88.7% | 91.8% |
| Gaussian Noise (σ=0.05) | 88.3% | 86.9% | 90.5% |
| Salt & Pepper (1%) | 89.7% | 87.5% | 91.3% |
| Contrast ×1.5 | 92.0% | 90.2% | 92.8% |

### Calibration

MesoNet++ achieves an **Expected Calibration Error (ECE) of 0.0089**, well below the 0.05 threshold for well-calibrated models.

### Grad-CAM Interpretability

For fake images, activation maps consistently focus on the **periocular (eye) and perioral (mouth)** regions — the areas where autoencoder bottlenecks in face-swap pipelines introduce characteristic blurring. Real images show diffuse, spatially inconsistent activations, confirming the model is not exploiting dataset-specific biases.

## Limitations

- Not yet evaluated on [FaceForensics++](https://github.com/ondyari/FaceForensics) or [DFDC](https://ai.facebook.com/datasets/dfdc/) benchmarks
- Cross-model generalization (e.g., StyleGAN2 → ProGAN) is not addressed by the current modifications
- Evaluation is on still images; video-level temporal aggregation is left for future work
